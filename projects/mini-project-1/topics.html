Arrays & Strings

Master scanning, in-place mods, and constant space. Fast wins in interviews & real-world streaming.

Two Pointers

When: Sorted arrays, palindrome, cycle detect.

  • Opposite ends: left=0, right=n-1
  • Fast/Slow: one reads, one writes
  • Floyd cycle detect
let w=1;
for(let r=1; r<nums.length; r++)
  if(nums[r]!==nums[r-1])
    nums[w++]=nums[r];
return w;

Sliding Window

When: Contiguous subarrays, max/min.

  • Fixed window: move together
  • Variable: expand right, shrink left
  • Encode invariant in ok()

Prefix/Suffix

  • Range sum in O(1)
  • Product except self
  • Max left/right precompute

Hashing

  • Two Sum (target - x)
  • Anagrams (sort or freq)
  • Consecutive seq (set)

Kadane (Max Subarray)

let max=nums[0], curr=nums[0];
for(let i=1; i<nums.length; i++){
  curr=Math.max(nums[i], curr+nums[i]);
  max=Math.max(max, curr);
}

Complexity

  • Two pointers: O(n), O(1)
  • Sliding: O(n), O(k)
  • Prefix: O(n)+O(1)
  • Hash: O(n), O(n)

Recursion & Backtracking

Systematically explore decision trees. Optimization, constraint satisfaction, and AI search.

Recipe

  1. Define state
  2. Base case (record solution)
  3. Generate choices
  4. Prune invalid
  5. Recurse & undo

Template

function backtrack(state){
  if (complete(state)){
    solutions.push(copy(state));
    return;
  }
  for (const c of choices){
    if (!valid(c)) continue;
    make(state, c);
    backtrack(state);
    undo(state, c);
  }
}

Subsets (2^n)

  • Include or exclude each element
  • With duplicates: sort, skip dups
  • O(n·2^n) time

N-Queens

  • Row-by-row placement
  • Track cols, diag1, diag2
  • O(N!) worst case

Pitfalls

  • Forget undo state
  • Shallow copy (use [...])
  • Dup handling: sort+skip
  • Off-by-one termination

Real-World

  • Sudoku, SAT solvers
  • Chess minimax
  • Traveling salesman
  • ML feature selection

Trees & Graphs

Traversal, connectivity, shortest paths. Feeds, maps, schedulers, game AI.

BFS

  • Shortest path (unweighted)
  • Multi-source expansions
  • O(V+E) time

DFS

  • Preorder, inorder, postorder
  • Cycle detection (colors)
  • Connected components

Dijkstra

  • Non-negative weights
  • Min-heap priority queue
  • O((V+E)logV)

Union-Find

  • Connectivity queries
  • Path compression + union by rank
  • Near O(α(n)) amortized

Real-World

  • Google Maps routing
  • Social graph clustering
  • Dependency resolution
  • Network flow analysis

Complexity

  • BFS/DFS: O(V+E)
  • Dijkstra: O((V+E)logV)
  • MST: O(ElogE)
  • Union-Find: O(α(n))

Dynamic Programming

Convert recursion to tabulation. State, transition, base case, fill order.

1D Knapsack

for(let w=W; w>=item.w; w--)
  dp[w]=Math.max(dp[w],
    dp[w-item.w]+item.val);

Key: Reverse loop for 0/1.

LIS (n log n)

const tails=[];
for(const x of nums){
  let i=lowerBound(tails,x);
  if(i===tails.length) tails.push(x);
  else tails[i]=x;
}

Common Patterns

  • Coin Change (Unbounded)
  • Edit Distance (Strings)
  • Matrix Paths (Grid)
  • Partition Subset

Pitfalls

  • Wrong loop direction
  • Overflow with sentinel values
  • Missing base init
  • State not minimal

Real-World

  • Sequence alignment (DNA)
  • Inventory mgmt (knapsack)
  • ML training
  • Diff algorithms

Complexity

  • 1D Knapsack: O(nW)
  • LIS: O(nlogn)
  • Edit Dist: O(mn)
  • Coin: O(n·target)

Sorting & Searching

Binary search templates, selection, and sorting variations under pressure.

Binary Search Lower

let lo=0, hi=n;
while(lo<hi){
  const mid=(lo+hi)>>1;
  if(pred(mid)) hi=mid;
  else lo=mid+1;
}
return lo;

Quickselect

  • Kth largest average O(n)
  • Pivot partition + recurse
  • Worst O(n²) rare

Heaps (Top-K)

  • Min-heap size k
  • O(nlogk) time
  • E.g., kth largest stream

Sorting Algos

  • Merge: O(nlogn) stable
  • Quick: O(nlogn) avg
  • Heap: O(nlogn) guaranteed

Pitfalls

  • Mid overflow: lo+(hi-lo)/2
  • Infinite loop: ensure pred changes
  • Off-by-one errors
  • Duplicates handling

Real-World

  • DB indexes (binary search)
  • Load balancing
  • Top products ranking
  • Median finding

Math & Bit Manipulation

Small tricks, big wins. Precomputation, modular arithmetic, bit masks.

Sieve (Primes)

const sieve=(n)=>{
  const p=Array(n+1).fill(true);
  for(let i=2; i*i<=n; i++)
    if(p[i]) for(let j=i*i; j<=n; j+=i)
      p[j]=false;
  return p;
};
